# Autonomous Multi-Agent Market Panel 🤖

An institutional-grade financial analytics dashboard that coordinates a specialized research panel using independent LLM personas, localized technical indicator algorithms, and strict structural JSON schema enforcement.

## 🛠️ System Architecture & Engineering Breakdown

The platform executes market research through a decoupled, sequential agent pipeline to eliminate framework overhead and guarantee data stability:

- **The Sentimentalist Persona**: Parses raw textual market news streams or fundamental articles using advanced natural language processing (NLP) to isolate macroeconomic triggers and classify current sentiment as BULLISH, BEARISH, or NEUTRAL.
- **The Quantitative Analyst Persona**: Runs a native mathematical momentum calculation over a comma-separated stream of historical asset closing prices, computing a trend direction and velocity metric.
- **The Chief Risk Officer Persona**: Collects the data vectors generated by both the Sentimentalist and Quantitative passes, cross-examines them for market contradictions, generates a strict financial risk score (1–5), and summarizes an actionable advisory digest.

## Core Engineering Highlights

- **Deterministic Schema Enforcement**: Leverages litellm and hardware response format variables to mandate a strict JSON output from the final agent, eliminating text hallucinations and string truncation issues.
- **Local Edge Hardware Acceleration**: Powered entirely offline using Ollama running Microsoft's and Qwen's highly optimized models (qwen2.5:3b) on consumer hardware (Ryzen 5 3600 CPU and Radeon RX 6600 GPU with 8GB VRAM).
- **Hybrid Cloud-Edge Architecture**: Implements run-time environment sniffing. The application detects if it is running on local hardware or hosted on Streamlit Cloud, seamlessly switching between local Ollama inference and the web-hosted Google Gemini API gateway.

## 📦 Installation & Local Configuration

### Prerequisite: Set Up Ollama to Run on an HDD/D-Drive

To conserve primary drive space, configure Ollama to save and run models from secondary storage:

1. Create a directory named `D:\OllamaModels`.
2. Open Windows Environment Variables and create a new User Variable:
   - Variable Name: `OLLAMA_MODELS`
   - Variable Value: `D:\OllamaModels`
3. Restart your terminal and download the required core engine:

```dos
ollama run qwen2.5:3b
```

### Application Setup

**Clone the Project Workspace:**

```dos
git clone https://github.com/yourusername/market-agent-panel.git
cd market-agent-panel
```

**Configure Environment Keys:**

Create a local `.env` file in the root folder to house your private backup tokens:

```
GEMINI_API_KEY=your_actual_api_key_here
```

**Install Python Packages:**

```dos
pip install -r requirements.txt
```

**Launch the Panel:**

```dos
streamlit run app.py
```
